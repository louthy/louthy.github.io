Effects are functorial, monadic, and applicative types that are designed to capture IO based side-effects.

Type Section Description
Aff<A> Aff Asynchronous IO. Captures asynchronous effects, handles exceptions elegantly.
Aff<RT, A> Aff Asynchronous IO. Captures asynchronous effects, handles exceptions elegantly, and has an injectable runtime (RT) which can provide configuration and dependency-injection
Eff<A> Eff Synchronous IO. Captures synchronous effects, handles exceptions elegantly.
Eff<RT, A> Eff Synchronous IO. Captures synchronous effects, handles exceptions elegantly, and has an injectable runtime (RT) which can provide configuration and dependency-injection
Proxy<RT, UOut, UIn, DIn, DOut, A> Pipes Base of all Pipes types - single type that unifies unidirectional bidirectional streaming channels.
Producer<RT, OUT, A> Pipes Producers yield. Derived from Proxy, opens the downstream-out, closes all other channels
Consumer<RT, IN, A> Pipes Consumers await. Derived from Proxy, opens the upstream-in, closes all other channels
Pipe<RT, IN, OUT, A> Pipes Pipes await and yield. Derived from Proxy, opens the upstream-in and downstream-out, closes all other channels
Client<RT, REQ, RES, A> Pipes Clients request. Derived from Proxy, opens the upstream-in and upstream-out, closes all other channels
Server<RT, REQ, RES, A> Pipes Servers respond. Derived from Proxy, opens the downstream-in and downstream-out, closes all other channels
Effect<RT, A> Pipes Fused composition of Producer, Pipe, and Consumer - or - Client, Pipe, and Server. Resulting in a Proxy derived Effect that that has no external channels open (the effect is 'closed', meaning it's sound - and can be run)
